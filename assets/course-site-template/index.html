Chapter 1

为什么需要注意力机制

通俗易懂的解释

可以把注意力机制理解成“读一句话时，当前词会主动去看哪些上下文最重要”。相比把整句压成一个固定长度向量，注意力机制允许模型在需要的时候重新读取相关位置。

底层原理

在经典 Encoder-Decoder 结构里，固定长度上下文向量容易形成信息瓶颈。注意力机制通过可学习的相关性打分，在解码的每一步都动态选择更有用的上下文表示。

数学原理深挖

最早的注意力可以抽象成对一组上下文向量做加权求和，其中权重来自打分函数后的归一化分布。直觉上，它是一个“软选择”过程，而不是硬检索。

常见误解

注意力不是“真正理解”，它只是让模型更灵活地分配信息读取权重。

这一章你该记住什么

注意力机制的核心价值，是打破固定向量瓶颈，让模型按需读取上下文。

Chapter 2

Self-Attention 怎么算

通俗易懂的解释

Self-Attention 的直觉是：序列中的每个 token 都会问自己一个问题，“在理解我时，句子里的其他位置谁最重要？”

底层原理

每个 token 会生成 Query、Key、Value。当前 token 的 Query 与其他 token 的 Key 做相似度计算，得到权重，再对对应的 Value 加权求和。

数学原理深挖

Attention(Q, K, V) = softmax((QK^T) / √dk) V

这里的缩放项用于控制内积随维度增大而变大的问题，避免 softmax 进入过于尖锐的区域。

常见误解

Q、K、V 不是三份完全不同的输入，而是同一份表示经过不同线性投影后的结果。

这一章你该记住什么

Self-Attention 本质上是“相似度加权汇总”，不是神秘黑盒。

复盘提示

Chapter 3

多头注意力为什么有效

通俗易懂的解释

多头注意力就像让多个“观察角度”并行看同一句话，有的关注语法关系，有的关注长距离依赖，有的关注实体对齐。

底层原理

多个 head 在不同投影子空间中分别做注意力，再把结果拼接起来，让模型同时保留多种关系模式。

常见误解

多头不是简单重复计算；关键在于不同投影空间允许 head 学到互补关系。

这一章你该记住什么

多头的价值在于表达多样性，而不是单纯增加计算量。
