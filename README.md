# StudySkill

一个专门用于“生成课程型知识网站”的 Codex skill。

它的目标不是只生成一个好看的页面，而是帮助你稳定产出真正适合学习、复习、做题和长期沉淀的知识课程网站。

仓库地址：<https://github.com/ArthurTood/studyskill>

## 这个 skill 解决什么问题

当你想把一个复杂主题做成学习网站时，常见问题通常是：

- 页面看起来像宣传页，不像课程页
- 只有概念解释，没有底层原理和数学支撑
- 图很多，但和知识点关系不大
- 结构不连续，需要频繁点切换
- 有练习题，但没有收藏题、错题回顾和本地保存

这个 skill 的设计目标是把这些约束直接固化下来，让每次生成的网站都更像“可学习的课程”，而不是“可浏览的介绍页”。

## 适用场景

当用户出现这些意图时，应该触发这个 skill：

- 生成讲解某一知识原理的网站
- 生成课程网站、学习网站、长文教程网站
- 基于 arXiv 论文生成知识讲解页面
- 生成题库版、考试版、刷题版学习网站
- 希望网站支持收藏题、错题回顾、本地保存

## 这个 skill 强制约束的输出风格

### 内容来源

优先使用：

- arXiv 论文
- 原始论文
- 官方文档
- 权威一手资料

不应只依赖营销内容、聚合文章或二手博客。

### 页面结构

页面必须是连续下拉阅读的课程型结构：

- 中间正文按章节顺序完整展开
- 用户只需要向下滚动即可连续阅读
- 左侧保留 sticky 章节总览
- 左侧只做导航，不负责切换正文

### 每章节固定内容

每个章节都必须包含：

- 通俗易懂的解释
- 底层原理
- 适当图解
- 数学原理深挖
- 常见误解
- 这一章你该记住什么
- 复盘提示
- 章节小测

整站最后还必须包含：

- 章节总测
- 索引
- 题库区

### 视觉方向

默认风格参考 Notion 式内容卡片页面，但更偏课程阅读：

- 背景：暖米色、柔和、干净
- 卡片：浅米白、大圆角、细边框、轻阴影
- 标题：更粗、更紧凑，有课程页感
- 正文：留白充足、长文舒适、层级清晰

避免：

- 花哨动效
- 过重装饰
- 与主题无关的配图
- 左侧点击切换正文

### 图解要求

图解必须服务于知识点本身，而不是装饰页面。

优先围绕这些内容绘制：

- 模型结构图
- 数据流图
- 模块关系图
- 关键机制拆解图
- 公式变量关系图

目标效果应更接近“教材插图”或“论文结构重绘”，而不是海报式配图。

### 题库能力

页面底部题库默认支持：

- 判分
- 收藏题
- 错题回顾
- 本地保存

如无额外要求，默认使用浏览器本地存储实现，例如 `localStorage` 或 `IndexedDB`。

## 使用方式

在 Codex 中直接这样调用：

```text
用 $studyskill 生成一个讲解 Transformer 原理的课程网站，面向有基础的前端工程师，偏直觉 + 数学，带题库版。
```

也可以这样：

```text
Use $studyskill to create a continuous-scroll course website that teaches diffusion models for beginners, grounded in arXiv papers, with diagrams, chapter quizzes, and local wrong-question review.
```

## 推荐提示词模板

你可以按下面这个格式直接提要求：

```text
用 $studyskill 生成一个讲解【主题】的课程网站，
面向【目标读者】，
深度偏【直觉 / 工程 / 数学 / 论文】，
是否需要题库版：【是 / 否】，
是否需要考试版：【是 / 否】，
页面风格保持连续下拉阅读，左侧只保留 sticky 导航。
```


## 关键文件说明

- `SKILL.md`：skill 的触发条件、执行流程和输出约束
- `references/course-site-spec.md`：课程型知识网站的稳定规范
- `agents/openai.yaml`：UI 元数据与默认调用提示

## 安全与质量原则

- 不直接注入不可信 HTML
- 优先引用一手资料与论文
- 图解要与知识点对应，避免误导
- 默认不引入无关统计或跟踪脚本
- 优先保证知识准确性、结构清晰度和学习效果
